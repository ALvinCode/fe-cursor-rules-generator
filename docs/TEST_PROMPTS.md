## 1. Boss 类系统页面生成示例

### 示例 1.1：数据表格管理页面（基础）

```
请创建一个用户管理页面，包含以下功能：

1. 页面布局：
   - 顶部：搜索栏（支持按用户名、邮箱、手机号搜索）
   - 中间：数据表格，显示用户列表
   - 底部：分页组件

2. 表格列：
   - 用户ID
   - 用户名
   - 邮箱
   - 手机号
   - 注册时间
   - 状态（启用/禁用）
   - 操作列（编辑、删除按钮）

3. 功能要求：
   - 支持按状态筛选（全部/启用/禁用）
   - 支持按注册时间排序
   - 点击编辑按钮弹出编辑对话框
   - 点击删除按钮显示确认提示
   - 表格支持多选，批量删除功能

4. 交互要求：
   - 搜索和筛选实时更新列表
   - 加载状态显示
   - 操作成功后显示提示消息
   - 删除操作需要二次确认

请使用项目中的组件库和工具函数，遵循项目的代码规范和架构设计。
```

### 示例 1.2：表单页面（中等复杂度）

```
请创建一个商品编辑页面，包含以下功能：

1. 页面结构：
   - 左侧：表单区域（70%宽度）
   - 右侧：预览区域（30%宽度，实时预览）

2. 表单字段：
   - 基本信息：
     * 商品名称（必填，最大50字符）
     * 商品分类（下拉选择，支持级联选择）
     * 商品描述（富文本编辑器）
     * 商品标签（多选标签，支持自定义添加）
   
   - 价格信息：
     * 原价（数字输入，保留2位小数）
     * 现价（数字输入，需小于原价）
     * 库存数量（整数输入）
     * 是否启用（开关）
   
   - 媒体信息：
     * 主图上传（支持拖拽，最多1张，支持预览）
     * 详情图上传（支持拖拽，最多9张，支持排序）
   
   - 其他设置：
     * 上架时间（日期时间选择器）
     * 排序权重（数字输入，默认0）

3. 功能要求：
   - 表单验证：实时验证，错误提示显示在字段下方
   - 预览区域：实时显示商品卡片样式
   - 保存按钮：保存到草稿（不验证必填项）或发布（完整验证）
   - 取消按钮：返回上一页，有未保存内容时提示确认

4. 交互要求：
   - 图片上传支持进度显示
   - 富文本编辑器支持图片插入
   - 表单字段联动：选择分类后自动加载该分类的属性配置
   - 自动保存草稿功能（每30秒）

请遵循项目的表单处理规范和组件使用规范。
```


## 3. 多国家线类系统页面生成示例

### 示例 3.1：多语言列表页（基础）

```
请创建一个支持多国家的商品列表页面，包含以下功能：

1. 国际化支持：
   - 页面顶部：语言切换器（支持中文、英文、泰文、印尼文）
   - 所有文本内容支持多语言切换
   - 日期、数字、货币格式化根据当前语言环境

2. 页面结构：
   - 顶部：搜索栏（占位符文本国际化）
   - 筛选栏：分类筛选（分类名称国际化）
   - 内容区：商品卡片列表
   - 底部：分页组件（分页文本国际化）

3. 商品卡片：
   - 商品名称（根据当前语言显示对应翻译）
   - 价格（根据国家显示对应货币符号和格式）
     * 中国：¥ 100.00
     * 泰国：฿ 100.00
     * 印尼：Rp 100.000
   - 销量显示（根据国家习惯格式化数字）
   - 标签文本国际化

4. 功能要求：
   - 语言切换后保持当前筛选条件
   - 语言切换后重新加载数据（获取对应语言的数据）
   - URL参数支持语言标识（如 ?lang=th）
   - 语言偏好本地存储

5. 数据格式：
   - 日期格式根据语言环境（中文：2024年1月1日，英文：Jan 1, 2024）
   - 数字格式根据语言环境（千分位分隔符不同）
   - 时区处理（根据国家显示对应时区时间）

请遵循项目的国际化规范和本地化处理规范。
```

### 示例 3.2：多语言表单页（中等复杂度）

```
请创建一个支持多国家的用户注册页面，包含以下功能：

1. 国际化支持：
   - 语言切换器（固定在顶部右侧）
   - 所有表单标签、提示、错误信息国际化
   - 验证规则提示国际化

2. 表单字段（根据国家显示不同字段）：
   - 通用字段：
     * 邮箱（必填，国际化验证提示）
     * 密码（必填，国际化强度提示）
     * 确认密码（必填，国际化匹配提示）
   
   - 国家特定字段：
     * 中国：姓名、身份证号（验证格式）
     * 泰国：ชื่อ（泰文姓名）、เลขบัตรประชาชน（身份证号）
     * 印尼：Nama、NIK（身份证号）
     * 手机号格式根据国家验证（中国11位，泰国10位等）

3. 表单验证：
   - 实时验证（错误提示国际化）
   - 字段级验证提示（显示在字段下方）
   - 提交前统一验证
   - 验证规则根据国家不同（如手机号格式）

4. 功能要求：
   - 国家选择器（影响显示的字段和验证规则）
   - 语言切换不影响已填写内容
   - 提交数据包含语言标识
   - 成功提示国际化

5. 交互要求：
   - 表单字段标签根据语言自动调整对齐方式（RTL支持）
   - 错误提示动画效果
   - 加载状态提示国际化
   - 网络错误提示国际化

6. 数据格式：
   - 日期选择器格式根据语言环境
   - 电话号码输入框根据国家显示不同占位符和格式
   - 表单提交数据包含locale字段

请遵循项目的国际化表单处理规范和验证规范。
```

## 📝 使用说明

### 测试流程建议

1. **准备阶段**：
   - 使用 `cursor-rules-generators` 为项目生成 Cursor Rules
   - 确认规则文件已正确生成在 `.cursor/rules/` 目录

2. **测试阶段**：
   - 选择一个 prompt 示例
   - 在 Cursor 中直接使用该 prompt
   - 观察 AI 生成的代码是否符合项目规范

3. **评估维度**：
   - **代码风格**：是否符合项目的代码风格规范
   - **架构设计**：是否遵循项目的架构设计
   - **组件使用**：是否正确使用项目中的组件和工具
   - **最佳实践**：是否遵循项目的最佳实践
   - **国际化处理**：多语言场景下是否正确处理国际化

4. **优化迭代**：
   - 根据测试结果调整 Cursor Rules
   - 补充缺失的规范说明
   - 优化规则描述使其更清晰

### 测试建议

- **从简单到复杂**：先测试基础示例，再测试复杂示例
- **覆盖不同场景**：确保测试不同类型的页面和功能
- **关注细节**：注意 AI 是否遵循了项目的细节规范
- **记录问题**：记录不符合预期的生成结果，用于优化规则

### 扩展示例

你可以基于这些示例，结合你的具体业务场景，创建更多针对性的测试 prompt。建议包含：

- 项目特定的业务逻辑
